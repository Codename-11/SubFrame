Add a new 'Workflows' tab to the existing GitHub panel that displays GitHub Actions workflows and their recent run statuses using the gh CLI. Users can see at a glance which workflows are passing, failing, or in progress, and click through to view runs on GitHub.

## Steps
- [ ] Define GitHubWorkflow and GitHubWorkflowRun interfaces and IPC channels in ipcChannels.ts
- [ ] Add loadWorkflows and loadWorkflowRuns methods to githubManager.ts using gh CLI
- [ ] Create useGithubWorkflows and useGithubWorkflowRuns hooks in useGithub.ts
- [ ] Build GithubWorkflowsPanel component with status badges and run list
- [x] Wire Workflows tab into GithubPanel tab bar and routing

## Acceptance Criteria
- New 'Workflows' tab appears in the GitHub panel alongside Issues, PRs, Branches, Worktrees, and Changes
- Lists all repository workflows with their name and current state
- Shows recent runs for each workflow with status badges (success/failure/in_progress/cancelled)
- Displays run metadata: branch, commit message summary, trigger event, duration, and timestamp
- Clicking a workflow run opens it on GitHub via shell.openExternal
- Loading and error states match existing GitHub panel tab patterns
- Refresh button fetches latest workflow data
- Gracefully handles repos with no workflows or when gh CLI is unavailable
