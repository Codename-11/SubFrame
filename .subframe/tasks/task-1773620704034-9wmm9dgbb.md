Extend SubFrame's Health/Overview panel into a full Agent Settings view where users can granularly include/exclude SubFrame components (hooks, knowledge, skills, plugins) during init and post-init, preview component contents in a detail view for transparency, and detect/manage global AI tool configuration files (claude global settings, CLAUDE.md, Gemini/Codex equivalents) as a central management hub.

## Steps
- [ ] Research and catalog all manageable SubFrame components and global AI config file paths across Claude, Gemini, and Codex
- [ ] Add IPC channels and main-process logic to detect, read, and write global AI tool config files
- [ ] Build Agent Settings panel UI with component toggle list (include/exclude) and detail preview drawer
- [ ] Integrate granular component selection into the project init flow (both UI dialog and CLI)
- [ ] Add global AI tool config viewer/editor section to the Agent Settings panel

## Acceptance Criteria
- Users can toggle individual SubFrame components (hooks, knowledge files, skills, plugins) on/off during project init
- Users can enable/disable individual components after init from a dedicated settings panel
- Each component (skill, plugin, hook, knowledge file) has a 'preview' action that opens a full detail view showing its contents
- SubFrame detects global AI tool config files (e.g. ~/.claude/settings.json, global CLAUDE.md, .gemini/, .codex/) and displays their status
- Users can view and edit detected global AI tool configs from within SubFrame
- Existing Health Panel functionality is preserved and accessible from the new view
- All new IPC channels are typed in ipcChannels.ts and follow existing patterns
