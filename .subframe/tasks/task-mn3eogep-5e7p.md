Research and design a unified notifications system for SubFrame. Currently GitHub notifications exist as a panel, but SubFrame could benefit from a global notification system that aggregates: GitHub notifications, sub-task status changes, workflow run completions/failures, agent session events, update availability, and other system events. Consider: notification center UI (bell icon in sidebar/titlebar with badge count), notification persistence (localStorage or file-based), notification preferences/filtering, toast vs persistent notifications, desktop OS notifications via Electron Notification API, and how this integrates with the existing per-panel notification sources.
